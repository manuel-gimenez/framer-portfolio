# Design process

Lightweight workflow for new pages or sections on this portfolio. Build in
Framer first; reach for code only when native can't do it
([`why-native-first.md`](../why-native-first.md)).

```
Research → Define structure → Execute → Review
```

| Phase | Goal |
| --- | --- |
| **Research** | Gather a few references; note what pattern you want (layout, not new tokens) |
| **Define structure** | Sketch section order and what is native vs code — on paper or in Framer |
| **Execute** | Build in Framer; import from [`02-components/`](../../02-components/) when needed |
| **Review** | Check mobile + desktop; tokens from [`01-foundations/`](../../01-foundations/) |

Skip phases for tiny changes (copy tweak, one interaction).
